## Summary

Set or clear link-to-previous behavior for a header/footer variant.

- Operation ID: `sections.setLinkToPrevious`
- API member path: `editor.doc.sections.setLinkToPrevious(...)`
- Mutates document: `yes`
- Idempotency: `conditional`
- Supports tracked mode: `no`
- Supports dry run: `yes`
- Deterministic target resolution: `yes`

## Expected result

Returns a SectionMutationResult receipt; reports NO_OP if link-to-previous already matches the requested value.

## Input fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `kind` | enum | yes | `"header"`, `"footer"` |
| `linked` | boolean | yes |  |
| `target` | SectionAddress | yes | SectionAddress |
| `variant` | enum | yes | `"default"`, `"first"`, `"even"` |

### Example request

```json
{
  "kind": "header",
  "linked": true,
  "target": {
    "kind": "section",
    "sectionId": "example"
  },
  "variant": "default"
}
```
